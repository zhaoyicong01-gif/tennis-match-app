# AceMatch 产品需求文档 (PRD)

## 1. 产品概述
**AceMatch** 是一款专为网球爱好者设计的智能社交应用。它通过标准化的水平评定、实时的空闲管理以及智能的匹配算法，解决网球运动中“找搭子难”、“水平不对等”的核心痛点。

---

## 2. 核心功能需求

### 2.1 水平评定系统 (NTRP Rating)
- **功能描述**：用户可以通过滑动滑块快速录入自己的 NTRP 等级（1.0 - 7.0）。
- **交互逻辑**：
  - 滑动时实时更新数值和对应的文字描述（如：初学者、中级、高级、专业）。
  - 该数值作为后续“智能匹配”的核心权重指标。

### 2.2 实时空闲日历 (Smart Availability)
- **功能描述**：提供一个以“周”为单位的视图，方便用户标记自己的打球空闲。
- **关键点**：
  - **实时性**：日期需根据当前系统时间自动生成，确保不跨年/月。
  - **交互**：今日日期高亮，未来日期显示“可预约”状态。
  - **目的**：通过看板形式直观展示本周打球计划。

### 2.3 智能搭子匹配 (Smart Matchmaking)
- **功能描述**：基于用户的 NTRP 等级和地理位置，推荐最合适的球友。
- **推荐指标**：
  - 水平接近度（NTRP 差值越小优先级越高）。
  - 地理位置（距离越近优先级越高）。
  - 匹配度百分比展示，增加社交透明度。

### 2.4 网球学院 (Academy & Tutorials)
- **功能描述**：集成高质量的网球教学视频，帮助用户提升技战术水平。
- **交互逻辑**：
  - 瀑布流展示教学卡片（封面图、时长、难度等级）。
  - 点击卡片弹出视频播放模式（支持内嵌播放，如 YouTube/Bilibili）。
  - 分类切换（如：正手、反手、发球、战术）。

### 2.5 社交打卡功能 (Check-in System)
- **功能描述**：激励用户记录训练过程，建立运动仪式感。
- **内容要求**：
  - 提供训练类型选择（底线对打、发球练习、正式比赛）。
  - 支持文字心得输入。
  - 手机端提供右上角悬浮按钮，支持全页面快速触发。

---

## 3. 非功能性需求

### 3.1 视觉与设计风格 (Aesthetics)
- **设计风格**：高端、现代、充满活力的 Dark Mode（暗黑模式）。
- **品牌色**：网球黄 (`#CEFF00`) + 球场绿 (`#2D5A27`)。
- **UI 特性**：
  - **Glassmorphism（玻璃拟态）**：使用毛玻璃效果增强层级感。
  - **微交互**：按钮悬浮阴影、页面淡入动画、卡片缩放反馈。

### 3.2 响应式适配 (Responsiveness)
- **Web 端**：左侧边栏导航，仪表盘网格布局。
- **Mobile 端**：
  - 侧边栏转换为**底部固定导航栏**。
  - Dashboard 布局自动切换为**垂直单列**。
  - 宽度适配所有主流机型尺寸。

### 3.3 部署与同步
- **CI/CD**：通过 Git + GitHub + Vercel 实现全自动部署。
- **性能**：静态资源最大化利用，视频采用 iframe 延迟加载以保证首页加载速度。

- [ ] **实时通讯**：增加球友私信聊天功能。
